# 双账号策划案 Skill

> 基于老孟编导知识体系，为餐饮/实体店客户输出完整的双账号策划案。内含老板个人号（信任账户）+ 门店号（转化引擎）。

---

## 核心理念

**一个问题→两个账号→一个闭环。**

餐饮实体店的核心问题永远是信任。乡镇/下沉市场尤甚——老百姓不认品牌，只认人。

双账号的本质分工：
- **老板号**：回答"这家店是谁开的？他图什么？我能不能信他？"——建信任，不卖货。
- **门店号**：回答"有什么菜？多少钱？值不值？什么时候去？"——降门槛，促转化。

---

## 一、触发条件

用户给你一个餐饮/实体店客户信息，要求做完整的短视频策划案。关键词：
"帮我给这个客户做账号策划"、"做个双账号方案"、"老板号怎么做+门店号怎么做"、"餐饮策划案"。

### 前提条件

必须有以下素材至少其一：客户采访记录、客户基本情况（地点/菜品/价格/老板背景）、区域市场信息。

**素材不全时，不猜。直接问用户要。**

---

## 二、六步工作流（不可跳步）

### Step 1：信息收集与事实确认

从素材中提取结构化事实：

| 维度 | 必须确认 |
|------|---------|
| 位置 | 城市/镇/街道，人口规模，经济特色 |
| 门店 | 桌数、定位（菜系）、阶段 |
| 老板 | 姓名、籍贯、履历、性格、真实动机 |
| 产品 | 特色菜、定价、套餐结构、回购数据 |
| 团队 | 厨师长、核心员工背景、管理风格 |
| 竞争 | 周边餐饮生态、消费者习惯、品类空缺 |

**红线：不编造任何数据。不确定的标"待确认"。**

### Step 2：十步决策链跑通盘

```
事实 → 目标 → 用户 → 选题 → 反应 → 形态 → 文案 → 声画 → 承接 → 复盘
```

**一次只打一个目标**（注意/信任/转化）。

**受众分层**（至少3-4类），每类给：一句话画像+核心心理屏障+内容策略。

典型分类：中层家庭（改善观望型）、本地有钱人（面子请客型）、生意人（低调谈事型）、年轻人（尝鲜社交型）。

**创作前四问**：烦什么？/ 什么话没人帮他说？/ 凭什么让他停？/ 什么对他算有用？

### Step 3：核心诊断（挖根）

五条诊断框架：
1. 产品问题 vs 信任问题
2. 真贵 vs 被认为贵（认知摩擦）
3. 有故事 vs 故事没讲
4. 品类认知（用户懂不懂你卖什么）
5. 场景机会（最大需求在什么场景）

商业五问：钱路在哪？→杠杆？→信息优势？→复购漏斗？→定价信号？

### Step 4：双账号战略设计

**老板号（信任账户）**：
- 人设四维度：身份标签/反差张力/价值观/说话方式
- 形态：观点型（引→破→立→证→界→钉）+故事型（异→困→择→价→变→断）
- 元素：反差、人群、怀旧、头牌
- 节奏：每周2-3条，不挂团购

**门店号（转化引擎）**：
- 定位：菜单翻译官、性价比证明人
- 形态：产品型（景→虑→异→证→舍→动）+过程型（动→问→处→果→断）
- 元素：成本、猎奇、反差、最差
- 节奏：每周5-7条，全挂团购

### Step 5：选题库构建

**老板号三大支柱**：
1. 我是谁/我为什么做这个（返乡/创业故事）— 怀旧·人群·反差
2. 我为什么这样做产品（专业理念）— 头牌·反差·成本
3. 我这样对待客户（信任系列）— 反差·人群

**门店号三大支柱**：
1. 套餐性价比（数字炸开认知）— 成本
2. 招牌菜展示（过程征服舌头）— 猎奇·反差·头牌
3. 顾客真实反应（社会认同）— 人群·最差

每个选题标注：编号+文案+核心观点+形态+元素+优先级(S/A/B/C)。

### Step 6：脚本撰写

```
【选题】【内容形态】【爆款元素】【目标反应】【钩子维度】
【钩子】首3秒原文
【正文】完整口语稿（分段，每段一个新信息）
【画面建议】
【自检】✓共振类型 ✓五关 ✓无禁忌开头
```

**老板号**：用老板自述Agent协议——去身份标签→真实情感入口→细节代替概括→展示矛盾。
**门店号**：用探店Agent协议——还原真实场景→抓最强烈感受→具体细节→诚实表达→拒绝广告腔。

**开头禁词**："今天来跟大家聊聊""最近我发现""大家好我是""很多人都在问"。
**内容红线**：不虚构、不编菜名价格、不吹牛、不承诺结果。

## 三、交付清单

1. 决策总纲：受众分层+决策快照
2. 市场透视：区域数据+消费结构
3. 客户透视：老板履历+核心差异点
4. 核心诊断：5条诊断+商业五问
5. 双账号战略：分工表+协同逻辑+阶段目标
6. 老板号：定位+人设+三大支柱+N选题+3-6条脚本
7. 门店号：定位+三大支柱+N选题+3-4条脚本
8. 执行体系：金句库+排期+五关质检+禁词+红线
9. 进阶：葫芦型定价+人生36宴

## 四、五关质检

| 关 | 标准 | 后果 |
|----|------|------|
| 虚构关 | 数据可核验或标"示例" | 毙 |
| 判断关 | 有条反常识主观点 | 毙 |
| 人话关 | 读出声12岁能懂 | 念一遍改到顺 |
| 细节关 | 数字>概念，具体>抽象 | 毙 |
| 行动关 | 结尾只有一个CTA | 毙 |

**三种共振**（至少中一才交付）：认知缺口("原来如此")/情感镜像("你懂我")/身份确认("我就是这样的人")

---

## 五、关键教训

1. **只写真实菜品**。客户素材有几个就写几个，不编不猜。
2. **套餐不编单品价**。不知道组成时只写总价+概括描述。
3. **原始选题最锋利**，后续迭代不要丢失。
4. **重要概念不丢**：葫芦型定价、人生36宴带出来。
5. **Agent协议是灵魂**，别用通用模板写脚本。
