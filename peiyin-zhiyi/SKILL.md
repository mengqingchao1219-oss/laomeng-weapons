# 🎙️ 配音之翼

> 把编导之刃写的文案，变成能直接合成MP3的口播脚本。含情绪、节奏、语气词，对接豆包TTS。

## TL;DR

- 🎯 **主任务**：把AI写好的文案处理成TTS就绪的口播脚本，指导用户完成音频合成
- ⚡ **一句话**：文案→梳合（口语化+抑扬顿挫+情绪标签+语气词+分段）→输出TTS脚本→用户一键合成
- 🔗 **技能联邦位置**：内容生产流水线最后一环。编导之刃出文案 → 配音之翼出音频
- 📦 **输出格式**：TTS口播脚本（含情绪指令+分段+语速音量）+ API接入指引

> **前置条件**：需要火山引擎账号 + 豆包语音API Key。没有的话，本文下半段有接入教程。

## 一、用户接入指南（首次使用必读）

### 第一步：开通火山引擎豆包语音服务

1. 打开 `https://console.volcengine.com/speech/`
2. 注册/登录火山引擎账号
3. 进入"语音合成" → 开通服务（有免费额度）
4. 进入 `https://console.volcengine.com/speech/new/setting/apikeys` 创建 API Key
5. 复制 `App ID` 和 `Access Token`

### 第二步：克隆自己的声音（可选但推荐）

1. 在豆包语音控制台 → "声音克隆" → "创建克隆音色"
2. 录制一段30秒以上的干音（安静环境、正常说话）
3. 提交后得到一串音色ID（格式如 `S_xxxxxxxxx`）
4. 把这个ID记下来——后面所有配音都用自己的声音

> **不用克隆也行**：豆包内置几十个高质量音色，直接挑一个用。

### 第三步：配置到本地

把你的 API 信息记下来，后面每次配音都会用到：

```
App ID:     你的App ID
Token:      你的Access Token  
音色ID:     你的克隆音色ID（或用内置音色）
Resource ID: seed-icl-2.0
```

### 第四步：安装配音脚本（可选，用于自动合成）

如果你要批量合成MP3，下载配套脚本：

```powershell
# 放到你项目的 scripts/ 目录下
pip install requests
python doubao_tts.py --help
```

> 脚本核心功能：读取TTS脚本 → 逐段调用豆包API → 输出MP3文件。本文末尾有脚本代码。

## 二、核心工作流：8步梳合流水线

当编导之刃产出文案后，自动进入以下流水线：

```
AI文案输入
  │
  ▼
Step 1: 判断内容类型 → 观点 / 故事 / 知识 / 过程 / 产品
  │
  ▼
Step 2: 口语化转化 → 书面→口语替换 + 开头禁词检查 + 长句拆分
  │
  ▼
Step 3: 抑扬顿挫 → 每句找信息峰 + [加重]/[放慢] 标签注入
  │
  ▼
Step 4: 语气词注入 → 嘛/呢/对吧/记住啊 等口语语气词（控制密度）
  │
  ▼
Step 5: 分段分角色 → 标注 hook/body/cta + 自动匹配情绪指令
  │
  ▼
Step 6: 参数匹配 → 语速、音量按场景自动设定
  │
  ▼
Step 7: 五关质检 → 虚构/没判断/不像人话/没细节/没下一步
  │
  ▼
Step 8: 输出TTS脚本 → 用户复制后一键合成MP3
  │
  ▼
输出：分段MP3文件 + 完整TTS口播脚本
```

### Step 2 详解：口语化转化规则

| 书面语 | → 口语 |
|--------|--------|
| 此外/另外/并且 | 删掉，直接说下一句 |
| 实现/达成/完成 | 弄成/做成/搞定 |
| 提供帮助 | 帮你 |
| 由于/因为……所以 | 删掉"由于"，"所以"换成"，你猜怎么着" |
| 不仅……而且 | 删掉，直接说两件事 |
| 长句（>30字） | 拆成两句，中间加停顿 |

**开头禁词（必删）**：
"今天来跟大家聊聊""最近我发现""大家好我是""很多人都在问""众所周知"

### Step 3 详解：抑扬顿挫规则

- `[加重]` — 数字、极端词、对比句结论 → 自动打重音标签
- `[放慢]` — 核心观点、关键结论、情感转折处 → 自动打放慢标签
- 每句自动注入 1-2 个标签，不每句都加

### Step 5 详解：情绪定义与磁场流动

#### 5.1 每种情绪的精确定义（避免模糊导致配音失真）

| 情绪名 | 这不是什么 | 这到底是什么 | 身体感觉 |
|--------|-----------|-------------|---------|
| 警告 | 不是喊叫、不是吓人 | 是把一件你不知道但你该知道的事，压低了告诉你。语速偏快但气息稳，像"我跟你说个事你别跟别人说" | 眉心微皱、身体前倾 |
| 揭秘 | 不是神秘兮兮、不是故弄玄虚 | 是"我发现了别人没发现的东西"，带着一点得意和急切想分享。声线压低但尾音上扬 | 嘴角微翘、像在说悄悄话 |
| 反差 | 不是故意反转、不是刻意设计 | 是"我之前也以为是A，结果发现是B"，前半句带着真实的疑惑（不是表演疑惑），后半句恍然大悟但不喊 | 前半：歪头/停顿→后半：点头/轻拍桌子 |
| 干货 | 不是讲课、不是说教、不是居高临下 | 是"这东西我真干过、真有效"，语气稳但别端着。像老匠人跟你聊他的手艺——他信，但不吹 | 坐着、手势自然 |
| 共鸣 | 不是煽情、不是卖惨、不是哭腔 | 是"我也经历过这个，我知道你什么感觉"。声音放低放慢，带一点疲惫但不说教。最怕演成"我也好惨"——你不是惨，你是走过来了 | 叹气感、声音有沙哑的边 |
| 自嘲 | 不是自贬、不是讨好 | 是"我知道这有点笨，但这就是我"。笑着说自己的弱点，但笑完不圆场。尾音可以飘一点，但不能虚 | 摇头+轻笑+继续说 |
| CTA | 不是命令、不是推销、不是喊口号 | 是"我说完了，你看着办"。短、干脆、不商量。像朋友说完重点，把杯子往桌上一搁。不是"你赶紧去"，是"我就说到这" | 身体坐正、语速快但字字清晰 |
| 留白 | 不是没说完、不是软绵绵 | 是"话说到这里，剩下的你自己想"。声量降到最低但不虚，最后几个字像往外吐气而不是往外喊。最忌"强行深情"——留白是轻的，不是重的 | 像对话结束前最后一句，声音散在空气里 |

#### 5.2 情绪磁场流动规则（禁止情绪单一平面化）

一条60秒的口播，情绪不能从头到尾一个调。必须按以下规则建立磁场流动：

```
钩子（0-5秒）：    警告/揭秘/反差 之一 —— 建立"心理缺口"
 ↓ 情绪转折点（第1次，约5-8秒处）
正文前半（5-25秒）：干货/共鸣 —— 填充信息 + 建立信任
 ↓ 情绪转折点（第2次，约25-30秒处）
正文后半（25-50秒）：共鸣→微微燃 —— 情感升级但不炸
 ↓ 情绪转折点（第3次，约50-55秒处）
结尾（50-60秒）：   CTA/留白 —— 收刀
```

**情绪转折点的切换信号**：
- 从"警告"切到"干货"的触发词：说完反常识判断后，第一句解释。如"为什么？因为……""你听我说……"
- 从"干货"切到"共鸣"的触发词：说完逻辑后，第一次提到"我"的经历。如"我见过……""我以前也……"
- 从"共鸣"切到"CTA"的触发词：说完感受后，第一次提到"你"。如"所以你要是……""你下次……"

**禁止的情绪组合**：
- 一条短口播（<30秒）情绪不超过2次转折——转多了像神经质
- 结尾留白前面不准接"警告"——情绪断崖会让观众不舒服
- 自嘲后面不准接"干货"——自嘲后直接给干货等于把自己的脆弱当工具，观众能感觉到
- 全片只有一种情绪 → 标记为"情绪磁场不合格"，必须重建

#### 5.3 情绪场景参数速查

| 场景 | 情绪指令 | 语速 | 音量 | 适用文案特征 |
|------|---------|------|------|------------|
| 钩子-警告 | [#压低声音、用紧张的警告语气说] | +10 | +20 | 有反常识判断、有数据冲击 |
| 钩子-揭秘 | [#凑近话筒、压低声音像说秘密一样说] | -5 | +10 | 有内幕感、"别人不说"的信息 |
| 钩子-反差 | [#先疑惑、后恍然大悟地说] | 变速 | +5 | 有"你以为A其实是B"结构 |
| 正文-干货 | [#沉稳自信、过来人的语气说] | 0 | 0 | 方法/经验/步骤/数据 |
| 正文-共鸣 | [#低沉真诚、带一点疲惫但坚定的语气说] | -10 | 0 | 有"我也经历过"的故事 |
| 正文-自嘲 | [#笑着摇摇头、有点无奈地说] | -5 | -5 | 有自我暴露、弱点展示 |
| 结尾-CTA | [#用命令的口吻、不容置疑地说] | +10 | +10 | 有行动指引、有决策点 |
| 结尾-留白 | [#轻声、像对自己说话一样说] | -15 | -10 | 有情感余味、不需要行动 |

**情绪指令公式**：
```
[#用{声线}的{情绪基调}、带{微情绪}的{语气风格}说]
```

### Step 6 详解：三层调控体系

| 层级 | 方式 | 控制什么 |
|------|------|---------|
| 基础声学层 | `speech_rate`(-50~100) / `loudness_rate`(-50~100) | 语速、音量 |
| 上下文语义层 | 文本措辞本身 | 情绪锚点 |
| 显式指令层 | `[#指令]`、`[动作描述]` | 直接指挥模型 |

## 四、与编导之刃的对接规则

1. 编导之刃产出文案后，自动进入配音之翼
2. 编导之刃的"拍摄建议"中如有画外音部分，直接提取为配音文案
3. 编导之刃的"字幕重点"标注处，自动标记为`[加重]`
4. 编导之刃的"记忆钉"处，自动标记为`[放慢]`+必读语气
5. 配音之翼处理完后，回传音频信息给编导之刃做"声画互补"检查

## 六、配音脚本模板（Python，可选）

以下为配套合成脚本的核心逻辑。保存为 `doubao_tts.py`，放在项目 `scripts/` 目录下：

```python
#!/usr/bin/env python3
"""豆包TTS配音合成脚本 - 配音之翼配套工具"""
import requests, json, os, sys

# ====== 用户配置区（替换为你自己的信息）======
APP_ID = "你的App ID"
ACCESS_TOKEN = "你的Access Token"
VOICE_ID = "你的音色ID"  # 克隆的音色ID，或内置音色
RESOURCE_ID = "seed-icl-2.0"
# =============================================

API_URL = "https://openspeech.bytedance.com/api/v3/tts/unidirectional"

def synthesize(text, output_path, speech_rate=0, loudness_rate=0):
    """调用豆包TTS API合成单段文本"""
    headers = {
        "Authorization": f"Bearer;{ACCESS_TOKEN}",
        "Content-Type": "application/json"
    }
    payload = {
        "app": {"appid": APP_ID, "token": "placeholder"},
        "user": {"uid": "doubao-tts-user"},
        "audio": {
            "voice_type": VOICE_ID,
            "encoding": "mp3",
            "speed_ratio": 0.0,  # deprecated, use resource.speech_rate
            "rate": 24000
        },
        "resource": {
            "resource_id": RESOURCE_ID,
            "speech_rate": speech_rate,
            "loudness_rate": loudness_rate,
            "format": "mp3"
        },
        "request": {
            "text": text,
            "text_type": "plain",
            "operation": "submit"
        }
    }
    resp = requests.post(API_URL, headers=headers, json=payload, stream=True)
    if resp.status_code == 200:
        with open(output_path, 'wb') as f:
            for chunk in resp.iter_content(chunk_size=8192):
                f.write(chunk)
        return output_path
    else:
        print(f"合成失败: {resp.status_code} {resp.text[:200]}")
        return None

def synthesize_all(segments, output_dir="output"):
    """批量合成所有分段"""
    os.makedirs(output_dir, exist_ok=True)
    files = []
    for i, seg in enumerate(segments):
        path = os.path.join(output_dir, f"seg_{i:02d}.mp3")
        result = synthesize(
            text=seg.get("text", ""),
            output_path=path,
            speech_rate=seg.get("speech_rate", 0),
            loudness_rate=seg.get("loudness_rate", 0)
        )
        if result:
            files.append(result)
            print(f"  ✓ {path}")
    return files

if __name__ == "__main__":
    print("配音之翼 - 豆包TTS合成工具")
    print("用法: python doubao_tts.py <文本或文件路径>")
```

## 七、五关质检

每条TTS脚本输出前，自动过以下五关：

1. **虚构关**：有没有编造的数据、不存在的细节？→ 删
2. **判断关**：有没有明确的观点判断？没有观点的配音=浪费流量
3. **人话关**：读出声像不像真人？有没有书面腔？→ 过口语基因
4. **细节关**：有没有至少一个具体细节能让观众"看见"？
5. **下一步关**：听完后观众知道该做什么吗？没有行动指引→补

## 八、运行规则

1. 首次使用提醒用户配API Key（见第一部分）
2. 用户提供文案后，按8步流水线执行，不可跳步
3. 口语化转化时，必须过开头禁词检查
4. 情绪标签只标注关键句，不每句都加
5. 五关质检发现问题必须修正后输出
6. 输出分两部分：A. TTS口播脚本 B. 合成指令
7. 如果用户没有克隆音色，推荐使用豆包内置音色（列出3-5个推荐音色ID）
8. 输出完后不再额外解释，直接给脚本
